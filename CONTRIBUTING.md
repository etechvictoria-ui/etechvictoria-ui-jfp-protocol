# Contributing to JFP

Thank you for your interest in contributing to JFP — JARO Flash Protocol.

JFP is an early-stage protocol for controlled AI-assisted development. Contributions are welcome, especially when they improve clarity, validation, examples, tooling, documentation, or real-world use cases.

## Project purpose

JFP is not a general prompt collection and not an uncontrolled AI generation framework.

The purpose of this project is to make AI-assisted work more structured, reviewable, and safer through:

- specification-first workflows
- explicit constraints
- output mapping
- validation checks
- SIGNAL_ORANGE halt conditions
- execution reports
- GitHub / CI integration examples

## How to contribute

1. Fork the repository.
2. Create a feature branch.
3. Make a focused change.
4. Add or update documentation when needed.
5. Add or update examples when the change affects behavior.
6. Open a pull request with a clear explanation.

Example branch names:

```text
feature/add-parser-example
fix/validator-section-check
Docs/update-quick-start
```

## Pull request guidelines

A good pull request should explain:

- what was changed
- why it was changed
- which files were intentionally modified
- whether any constraints or examples were updated
- how the change can be checked

Please avoid unrelated changes in the same pull request.

## JFP contribution rules

When adding or changing `.jfp` examples:

- define the request clearly
- include an OUTPUT_MAP
- include constraints
- include SIGNAL_ORANGE triggers where relevant
- avoid vague or uncontrolled behavior
- include an expected output or execution report when possible

## Documentation rules

Documentation should be written in clear English.

Every major file or example should explain:

- what it does
- why it exists
- how to run it
- what output to expect

## Commercial and larger-project use

JFP is released under the MIT License.

This means the project can be used freely, including in commercial environments, as long as the license notice is preserved.

If a company, larger project, or professional team wants support, integration help, custom workflow design, consulting, training, sponsorship, or attribution-based collaboration, they should contact the project maintainer directly.

Suggested collaboration paths:

- sponsored development
- paid integration support
- consulting around controlled AI-assisted workflows
- custom JFP templates for internal teams
- enterprise documentation or training
- attribution in papers, articles, demos, or products

Open-source use is welcome. Commercial support and collaboration can be discussed separately.

## Code of conduct

Be respectful, constructive, and specific.

Disagreements are acceptable. Personal attacks, harassment, spam, or abusive behavior are not.
