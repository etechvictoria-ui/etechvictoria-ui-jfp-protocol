# JFP Case Studies

This section presents example systems and application patterns that demonstrate how JFP ideas can be used in real workflows.

The purpose of this section is not to publish full production systems, but to show practical use cases, design patterns, and controlled AI-output structures.

## Included case studies

| Case study | Description |
|---|---|
| `viki-vision-ueact` | Visual decision-support prototype using a JFP-style output stream and human-in-the-loop review |

## Why case studies exist

JFP is not only a text format or a prompt pattern.

It is a protocol mindset for controlling AI-assisted work through:

- explicit facts
- separated decision layers
- output mapping
- quality checks
- correction layers
- uncertainty exposure
- final review

Case studies show how these concepts can appear in existing systems.

## Safety note

Case studies are provided for educational and architectural purposes.

They should not be interpreted as ready-to-use emergency, security, military, medical, or autonomous decision systems.
